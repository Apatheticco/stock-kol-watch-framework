# 账号名册 — Account Roster（方法论模板）

> **框架不预设任何账号。** 这是一份"如何建、如何评、如何养"你自己关注名单的方法论。
> roster 是你的情报网——质量取决于你选谁、怎么持续校准。下面给出 rubric 和模板，**真实账号由你填**。

---

## 一、怎么建 roster

**目标**：8-15 个**互补**的账号，覆盖你关心的几条主线，且**自带认知冲突**（有多有空，不是清一色看多）。

选账号时按这几个维度配齐，避免同质化：

| 维度 | 你要的 | 反面（避免） |
|------|--------|------------|
| **数据深度** | 带数据点 + 来源（财报/行业机构/制程数据）| 纯喊单无依据 |
| **视角多样** | 分析师 / 交易员 / 卖方聚合 / 架构师 / 宏观，各来一两个 | 全是同一类 |
| **多空平衡** | **至少 1-2 个稳定唱空/质疑的硬声音** | 全员 long-biased（最危险）|
| **可验证** | 观点能交叉验证 / 被其他高质量账号引用 | 黑箱、自说自话 |
| **活跃度** | 稳定产出 | 长期潜水 |

⚠️ **最常见的结构性缺口**：roster 不知不觉变成全员看多。**主动保留唱空/质疑的账号**，哪怕你不认同——它是你持仓 thesis 的"内部对照组"。

---

## 二、信号质量分级 rubric

| 级别 | 标准 |
|------|------|
| **A+** | 每条都带数据 + 来源，可立即据此做研究方向决策；必读 |
| **A** | 硬科普或硬数据，但可能慢半拍 |
| **B+** | 有交易动作或主题催化剂，但难直接复制 |
| **B** | 宏观框架价值，不直接对应标的 |
| **B-** | 事件记录为主，分析浅 |
| **C+** | 单条深度命中，其余噪音多 |
| **C / D** | 废话比例 > 70%（连续观察到这水平 → 考虑移除）|

**评级铁律**：
- **评的是信号质量，不是"他喊的票涨没涨"**（不评胜率）。
- **日频不动评级**——只累积事件。**周度才按质量复盘评级**，且变动需你自己确认。

## 三、画像深化（"评级是几分，画像是怎么用"）

每个账号除了评级，维护一份**怎么用**的画像。5 个维度，**每周用累积观察精修**（不是每周重写）：

| 维度 | 含义 |
|------|------|
| **独特 alpha** | 别处拿不到的具体能力（不是泛泛"半导体"，要具体到"制程良率第一落点"这种）|
| **风格 / tell** | 信号长什么样、习惯（RT 谁 / 长文 vs 短评 / 情绪化与否）|
| **盲区 / 打折项** | 偏见 / 利益相关（持仓推票 = talking his book）/ 立场漂移 / 可验证性低 → **哪些话要打折** |
| **最佳用法** | 在你决策里怎么用（leading indicator / 反方 cross-check / 催化 spotter / 情绪温度计…）|
| **关系网** | 与谁同源 / 对立 / 互引 |

**画像深化表模板**（放本文件顶部，每周 review 精修）：

| 账号 | 评级 | 独特 alpha | 盲区/打折 | 最佳用法 |
|------|------|-----------|----------|---------|
| @____ | _ | ____ | ____ | ____ |

⚠️ **同源警告**：若某账号同时是你的实盘仓位源（在别处跟踪其真实仓位），推文 + 实盘是**同一人两个 channel**，信号计数**只算 1 个**，不要重复加权。

⚠️ **认知冲突对**：roster 内对同一标的强多 vs 强空的两个账号 = **最有价值的认知冲突**，单独标注、成对追踪、不许只看一边就定调。

## 四、单账号画像模板

```markdown
### @<handle>（<显示名>）— <评级>（<入 roster 日期>）
- **画像**：<定位 + 风格，1-2 句>
- **入 roster 触发**：<为什么加他>
- **独特价值（填补什么缺口）**：<具体能力>
- **关注领域**：<标的 / 主题>
- **看的时候要注意（打折项）**：<偏见 / 利益相关 / 可验证性>
- **本期关键信号**：<带日期 + 链接的事件，每周累积>
```

## 五、何时增删 roster

- 某账号连续 3 次跑都是 C 级 → 提议移除
- 用户主动提到新账号 → 询问是否加入
- 关注领域大幅漂移（如从股票转纯加密）→ 提议移除/重分类
- 某账号连续 4 周 0 相关观点 → 周报时提议是否移除
- **不归档保留触发**：仍背书用户持仓 thesis / 仍代表板块独立 voice / 仍是实盘源

## 六、RT/QT 候选发掘（喂 Watchlist/Candidate-Roster.md）

每次跑日报扫 roster 的 RT/QT 网络，把被引用的**外部账号**累积成候选池：
- 记录：被谁引用 / 引用次数（7 天累积）/ 主题 / 典型示例
- **升级 trigger**：被 ≥2 roster KOL 引用 ≥3 次/7 天 → 提议加入；A+ 账号专门点名"alpha 贡献者" → 立即评估
- **排除**：已在 roster / 段子私人 / 与股票无关 / 默认头像 0 粉 / 机构官方账号
- **降级**：连续 2 周 0 引用 → archive

## 七、事件日志（每周末追加）

每周末跑周报时，按账号追加过去 7 天关键事件：

```
| 周次 | 关键事件 / thesis | 涉及标的 | 当时价格 | 原推链接 |
```

**核心原则**：不评胜率 / 不判断准确率 / 不靠它升降评级——**只累积事件**。长期 10+ 周累积后自行回看模式。
